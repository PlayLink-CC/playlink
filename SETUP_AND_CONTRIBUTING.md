# PlayLink - Setup & Contribution Guide\n\n## Getting Started\n\n### Prerequisites\n- Node.js 16.x or higher\n- npm or yarn package manager\n- Git version control\n- Backend server running on `http://localhost:3000`\n\n### Installation Steps\n\n1. **Clone the repository**\n   ```bash\n   git clone https://github.com/PlayLink-CC/playlink.git\n   cd playlink\n   ```\n\n2. **Install dependencies**\n   ```bash\n   npm install\n   ```\n\n3. **Create environment file**\n   ```bash\n   cp .env.example .env\n   ```\n   \n   Update `.env` with your configuration:\n   ```\n   VITE_API_BASE_URL=http://localhost:3000/api\n   VITE_APP_NAME=PlayLink\n   ```\n\n4. **Start development server**\n   ```bash\n   npm run dev\n   ```\n   \n   Application will be available at `http://localhost:5173`\n\n5. **Verify setup**\n   - Open browser to `http://localhost:5173`\n   - Check that app loads without errors\n   - Verify API connectivity in Network tab\n\n### Backend Setup\n\nMake sure the backend API is running:\n\n```bash\n# In backend repository\ncd playlink-backend\nnpm install\nnode server.js\n# Backend should be running on http://localhost:3000\n```\n\n## Development Workflow\n\n### Available Commands\n\n```bash\n# Development server with hot reload\nnpm run dev\n\n# Build for production\nnpm run build\n\n# Preview production build\nnpm run preview\n\n# Run linter\nnpm run lint\n\n# Fix linting issues automatically\nnpm run lint -- --fix\n```\n\n### Project Structure\n\n```\nplaylink/\nâ”œâ”€â”€ src/\nâ”‚   â”œâ”€â”€ main.jsx              # Brief description\n * @component\n * @param {Object} props\n * @returns {JSX.Element}\n */\nconst ComponentName = ({ prop1, prop2 }) => {\n  // State\n  const [state, setState] = React.useState(null);\n\n  // Effects\n  React.useEffect(() => {\n    // Setup\n    return () => {\n      // Cleanup\n    };\n  }, []);\n\n  // Handlers\n  const handleEvent = () => {\n    // Handle event\n  };\n\n  // Render\n  return (\n    <div>\n      {/* JSX */}\n    </div>\n  );\n};\n\nexport default ComponentName;\n```\n\n### Component Best Practices\n\n1. **Keep components small and focused**\n   - One responsibility per component\n   - Extract complex logic into separate functions\n\n2. **Use functional components**\n   - Use hooks (useState, useEffect, useContext)\n   - Avoid class components\n\n3. **Props over state**\n   - Pass data via props when possible\n   - Keep state local to component\n\n4. **Error boundaries**\n   - Add error handling in async operations\n   - Show user-friendly error messages\n\n5. **Performance optimization**\n   - Use React.memo for expensive components\n   - Memoize callbacks with useCallback\n   - Memoize values with useMemo\n\n## Git Workflow\n\n### Branch Naming\n\n```\nfeature/feature-name        # New feature\nfix/bug-description         # Bug fix\ndocs/documentation-topic    # Documentation\nrefactor/component-name     # Code refactoring\n```\n\n### Commit Message Format\n\n```\n<type>: <subject>\n\n<body>\n\n<footer>\n```\n\n**Types**: feat, fix, docs, style, refactor, test, chore\n\n**Example**:\n```\nfeat: add venue search functionality\n\nImplement search form component with axios integration.\nAdds ability to search venues by sport and location.\n\nCloses #123\n```\n\n### Pull Request Process\n\n1. Create Pull Request with description\n   - What changes were made\n   - Why changes were made\n   - How to test changes\n\n5.
